# Influence map

This starter intentionally blends several public patterns:

- a root orchestration skill with one canonical research packet
- linked skills that hand artifacts to downstream skills
- composable and mandatory stage workflows
- markdown agent personalities with workflows and success metrics
- hierarchical context injection and conditional rules
- skill-embedded tool bindings for web, docs, and code search
- adaptive web extraction as a capability rather than a hard-coded provider assumption

## What we keep from the original `research-pi`

- thorough research and comparison as two first-class modes
- canonical output model
- multiple packaging modes
- web-first and scholar-first evidence gathering
- explicit source evaluation and uncertainty handling

## What we change

- split large logic into linked atomic skills
- move provider and tool bindings into manifests and host config
- add graphable manifests beside `SKILL.md`
- treat agents as first-class linked nodes
- add a graph studio so scale does not become opacity
